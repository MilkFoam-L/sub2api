## 验证方式

- Git：`git status`、相关 `git diff`、分支/远端/备份分支存在性、提交日志。
- 残留搜索：独立调度路由、handler、API、DTO、优先账号、credential strategy/fallback、upstream-rate 采集模块、前端菜单/i18n key。
- 后端：优先运行相关 Go 单测，再运行可承受范围内的 `go test ./...`、`go build` 或仓库 Makefile 目标。
- 前端：项目现有 lint、TypeScript/Vue 类型检查、生产构建；检查路由、侧栏和设置页。
- 文本变量：搜索被删除页面的 i18n 引用和孤儿键，运行现有 i18n/typecheck 校验；必要时用浏览器检查活跃管理页面。
- 合并：核验 `v0.1.149` tag、合并基点、冲突文件和最终 diff。
- Docker：本地构建成功，必要时启动容器做健康检查；推送后读取远端 digest/manifest。

## 风险与应对

1. **误删原有调度能力**：以功能引入前历史和调用链为依据，只删新增模块；OpenAI 与普通 gateway 调度分别验证。
2. **工作树混有用户改动**：逐文件读 diff，不回滚、不全量格式化；提交前只暂存本目标文件。
3. **数据库破坏**：默认不执行生产 DROP 或破坏性迁移；无法证明安全时保留闲置表，代码层彻底停用。
4. **上游合并冲突**：先备份分支和清理提交，再用普通 merge；按功能边界迁移，不使用破坏性 Git 命令。
5. **文本变量缺失**：同时检查路由、菜单、Vue 模板、TypeScript 类型和 locale；构建与活跃页面验证双重确认。
6. **远端推送失败**：检查跟踪分支和权限，保持本地提交与备份分支完整；不强推。
7. **腾讯云镜像目标不明确**：优先沿用仓库和本机已存在的目标；不猜测生产命名、不泄露凭据。若环境客观缺少认证，完成可独立工作并把该步骤标记为有证据的 blocked。
8. **构建资源或网络失败**：记录完整错误，定位根因后重试或改用仓库等价验证入口；不虚报成功。

## 回滚方案

- 删除阶段：通过目标开始前的 Git 基线和原子提交进行 `git revert`，不使用 `reset --hard`。
- 合并阶段：使用合并前备份分支恢复，或对合并提交执行 `git revert -m 1 <merge-commit>`。
- 推送阶段：若远端代码有问题，创建回滚提交并正常推送。
- 镜像阶段：保留推送前稳定镜像标签/digest；部署回滚到上一稳定 digest，不覆盖或删除远端历史镜像。
